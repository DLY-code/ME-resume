## 4. 你必须会讲的基础概念

## 4.1 目标检测通用结构

`backbone`：负责提取图像特征  
`neck`：做多尺度特征融合，常见如 FPN、PAFPN  
`head`：输出分类和边界框

面试里如果问“某模型的 head / backbone 是什么”，至少先按这三层说。

## 4.2 anchor-based 和 anchor-free

`anchor-based`：

1. 先预设很多不同尺寸和长宽比的候选框
2. 模型学习的是在这些候选框基础上的偏移和分类
3. 代表：YOLOv5

`anchor-free`：

1. 不预设 anchor
2. 直接预测目标中心、边界距离或关键点
3. 代表：YOLOX、YOLOv8

差异：

1. anchor-based 依赖先验框设计，超参数较多
2. anchor-free 结构更简洁，迁移更方便

## 4.3 YOLOv5、YOLOX、YOLOv8 怎么说

### YOLOv5

1. 经典单阶段检测器
2. anchor-based
3. 工程成熟、生态完善、上手快

### YOLOX

1. anchor-free
2. 使用 decoupled head，把分类和回归分开
3. 样本分配策略更先进，训练表现更好
4. 精度和速度平衡不错

### YOLOv8

1. 也是 anchor-free
2. 整体训练和头部设计进一步优化
3. 使用体验更现代，适配任务更丰富

一句话总结：

“YOLOv5 更成熟易用，YOLOX 和 YOLOv8 更偏新一代 anchor-free 检测框架，结构更简洁，样本分配和训练策略也更先进。”

## 4.4 DETR / DINO 要会到什么程度

### DETR 核心架构

1. backbone 先提取特征
2. 将特征图展开成一串 token
3. 加上位置编码送入 transformer encoder-decoder
4. decoder 里有一组 object queries
5. 每个 query 负责预测一个目标的类别和框

### object query / token 是什么

1. token 可以理解为图像特征序列中的一个元素
2. object query 是一组可学习的查询向量，相当于“候选目标槽位”
3. `num_queries` 大致表示最多预测多少个目标

### DINO 要点

1. 属于 DETR 改进系列
2. 重点是 denoising 训练，帮助模型更快收敛、更稳定
3. 如果结合 Swin，就是用 Swin Transformer 做 backbone

## 4.5 DeepSORT 和 ByteTrack

### DeepSORT

1. 检测器先给出目标框
2. 卡尔曼滤波预测目标下一时刻位置
3. 用匈牙利算法做轨迹和检测框匹配
4. 除了运动信息，还会用外观特征 ReID embedding 辅助关联

一句话：

“DeepSORT 是运动信息和外观信息结合的多目标跟踪方法。”

### ByteTrack

1. 也先基于检测结果做关联
2. 核心特点是把低分检测框也利用起来
3. 先用高分框匹配已有轨迹，再用低分框补充未匹配轨迹
4. 对遮挡、置信度波动场景更友好

一句话：

“ByteTrack 的关键点是高低分检测框分阶段关联，不轻易丢掉低分框。”

## 4.6 分布式训练和普通训练

普通训练：

1. 一般单卡训练
2. batch size 和速度都受单卡限制

分布式训练：

1. 多卡并行训练
2. 每张卡处理一部分数据
3. 各卡算完梯度后做同步，再更新参数

常见影响：

1. 训练速度更快
2. 总 batch size 变大
3. 往往需要同步 BN、分布式采样器，并重新调学习率

## 4.7 样本分布不均匀怎么处理

先判断是哪种不均衡：

1. 类别不均衡
2. 正负样本不均衡
3. 某些场景或难样本占比不足

常见方法：

1. 重采样：过采样少数类、欠采样多数类
2. 数据增强：对少数类做更有针对性的增强
3. 损失函数加权：class weight、focal loss
4. 样本分配优化：让难样本、少数类样本更容易被学习到

面试时可以答：

“我会先判断是不均衡发生在类别层面还是检测正负样本层面，再分别从重采样、针对性增强、损失函数加权和样本分配策略几个方向处理。”

## 4.8 部署优化怎么答

你目前真实做过的是：

1. PyTorch 转 ONNX
2. ONNX 转昇腾 OM
3. 处理算子不兼容
4. 做精度对齐
5. 在边缘设备上完成推理验证

再往下一层，面试里可以补充“我知道但做得没那么深”的优化方向：

1. ONNX 图优化：常量折叠、算子融合、图简化
2. 精度优化：FP16、INT8 量化
3. 输入优化：固定 shape、减少数据拷贝
4. 后处理优化：NMS 加速、前后处理并行
5. 工程优化：batch、异步流水线、warmup

稳妥说法：

“我项目里实际做得最深的是模型转换、算子兼容和精度对齐。更系统的推理优化我也了解，通常会从图优化、量化、内存和后处理几个方向继续做。”
