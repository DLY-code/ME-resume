# 算法岗算法基础笔记

用途：这份笔记偏“入门解释版”，目标是让基础不扎实的人也能读懂；同时尽量保持排版紧凑，适合打印。

## 1. 目标检测器的通用结构

大多数目标检测模型都可以拆成三部分：`backbone + neck + head`。

- `backbone`
  是什么：负责从原始图像里提取特征。
  底层逻辑：原图像像素太原始，模型需要先把“边缘、纹理、形状、语义”逐层抽出来，后面才能判断哪里有目标。
  通俗理解：像先让模型“看懂图片”。
  常见：`ResNet`、`CSPDarknet`、`Swin Transformer`、`MobileNet`

- `neck`
  是什么：负责融合不同层级、不同尺度的特征。
  底层逻辑：浅层特征分辨率高，适合看小目标和细节；深层特征语义强，适合看大目标和整体类别。检测任务需要把这些信息拼起来。
  通俗理解：把“看细节的能力”和“看整体的能力”融合起来。
  常见：`FPN`、`PAN`、`PAFPN`

- `head`
  是什么：负责输出最终检测结果。
  底层逻辑：前面提到的是特征，最后还要把特征变成“这个位置是什么类、框在哪里、置信度多高”。
  通俗理解：把“看懂了什么”翻译成“具体答案”。
  常见输出：分类、框回归、objectness、IoU

- 一句话总结
  `backbone` 提特征，`neck` 融合特征，`head` 出结果。

## 2. anchor-based 和 anchor-free

这是检测岗位最爱问的基础概念之一。

- `anchor` 是什么
  `anchor` 可以理解成一批预先放好的“候选框模板”。比如在每个位置先放几个不同大小、不同长宽比的框，然后让模型判断这些框里有没有目标，以及框该如何微调。

- `anchor-based`
  是什么：先定义 anchor，再预测相对这些 anchor 的偏移量。
  底层逻辑：先给模型一个大致起点，再让它在这个起点上修正位置。
  优点：有先验，早期很多方法都靠它取得了不错效果。
  缺点：依赖 anchor 设计；换数据集时有时还得重新调 anchor 尺寸。
  常见模型：`Faster R-CNN`、`YOLOv3`、`YOLOv5`

- `anchor-free`
  是什么：不预设 anchor，直接预测目标位置。
  底层逻辑：让模型自己学习哪里是目标中心、框的上下左右边界在哪里，而不是先套模板。
  优点：结构更简洁，超参数更少，迁移到新任务时更省事。
  缺点：对训练策略、样本分配往往更敏感。
  常见模型：`FCOS`、`YOLOX`、`YOLOv8`

- 两者差别怎么记
  `anchor-based`：先给模板，再修框。
  `anchor-free`：不靠模板，直接找目标。

- 面试快答
  anchor-based 会先预设候选框，再学习偏移；anchor-free 则直接预测目标位置。前者先验更强，后者结构更简洁。

## 3. YOLOv5、YOLOX、YOLOv8 的差别

- `YOLOv5`
  特点：成熟、资料多、工程生态好、上手快。
  一般认知：常归为 `anchor-based`。
  适合怎么讲：它更像工业里很好用、很好落地的一代方案。

- `YOLOX`
  特点：`anchor-free`，并使用 `decoupled head`。
  `decoupled head` 是什么：把分类和框回归拆成两个分支，减少两种任务互相干扰。
  底层逻辑：分类关心“是什么”，回归关心“在哪里”，这两种目标不完全一样，拆开做通常更稳定。
  适合怎么讲：YOLOX 在结构和训练策略上更现代。

- `YOLOv8`
  特点：也偏 `anchor-free`，整体训练和任务支持更统一。
  适合怎么讲：它是更现代、更强调易用性的 YOLO 系列方案。

- 最实用的比较说法
  `YOLOv5` 更成熟、工程化更强；`YOLOX/YOLOv8` 更偏新一代 `anchor-free` 检测器，设计更简洁、训练策略也更先进。

## 4. DETR 系列：encoder / decoder / token / object query

很多人听到 DETR 就慌，其实先抓住主线就行。

- DETR 想解决什么问题
  传统检测器通常有很多手工设计：anchor、候选框、NMS、复杂匹配规则。DETR 想用 Transformer 的方式，直接把检测看成“输出一组目标”的问题。

- DETR 的主流程
  图像输入 -> `backbone` 提特征 -> 特征图展平成一串 `token` -> 加位置编码 -> `encoder` 做全局建模 -> `decoder` 接收一组 `object query` -> 每个 query 输出一个类别和框。

- `token` 是什么
  在这里可以把 `token` 理解成“视觉序列里的一个元素”。它可以是一块 patch，也可以是特征图上的一个位置。
  通俗理解：本来图片是二维的，Transformer 更擅长处理序列，所以先把图像特征改造成一个“序列”。

- `encoder` 是什么
  是什么：对整张图的特征做全局信息建模。
  底层逻辑：让图像中不同位置彼此“看到对方”，从而理解上下文关系。
  通俗理解：让模型知道“这个区域和别的区域有什么关系”。

- `decoder` 是什么
  是什么：根据 query 去编码后的特征里找目标。
  底层逻辑：encoder 已经准备好了整张图的信息，decoder 再拿着一组“查询槽位”去问：这里有没有目标？是什么？在哪里？
  通俗理解：像拿着很多个“候选岗位”去图里招人。

- `object query` 是什么
  是什么：一组可学习的向量。
  底层逻辑：每个 query 试图负责一个目标；如果没匹配到真实目标，就输出空结果或背景。
  通俗理解：像一组“目标座位”，每个座位尝试坐进一个目标。
  例如：`num_queries = 100`，可以理解为模型最多准备了 100 个“候选目标位”。

- DETR 的优缺点
  优点：结构优雅、端到端、减少手工设计。
  缺点：收敛慢、训练成本高、早期版本对小目标不够友好。

- 为什么有人说 DETR 不需要 NMS
  因为它本来就是“输出一组目标”的集合预测思路，不像传统检测器那样容易产生大量重复框，所以后处理上更简化。

## 5. DINO 和 DINO-Swin

这里说的是目标检测里的 `DINO`，不是自监督学习那个 DINO。

- `DINO` 属于什么
  它是 `DETR` 系列的改进版本。

- 关键改进怎么记
  最重要先记一句：`DINO` 的核心改进之一是 `denoising training`。

- `denoising training` 是什么
  通俗说，就是训练时故意给一些“带噪声的目标提示”，让模型更快学会怎么定位和匹配目标。
  底层逻辑：DETR 类模型一个难点是训练初期不容易学会“哪个 query 对应哪个真实目标”，DINO 通过加入带噪声的训练信号，让这个过程更稳定。

- `DINO-Swin` 是什么
  就是 `Swin Transformer` 做 `backbone`，`DINO` 做检测框架。
  通俗理解：前面用 Swin 提更强的特征，后面用 DINO 的 Transformer 检测逻辑来出结果。

## 6. 训练基础：损失函数

- 损失函数是干什么的
  它的作用是衡量“模型预测得有多差”，然后把这个差距传回去更新参数。

- 为什么检测里常常有多个损失
  因为检测不是一个单一任务，而是至少包含两件事：
  1. 这个框是什么类别
  2. 这个框的位置对不对

- 常见损失
  分类损失：`Cross Entropy`、`Focal Loss`
  回归损失：`L1`、`Smooth L1`、`IoU Loss`、`GIoU/DIoU/CIoU`

- `Focal Loss` 为什么常被问
  因为检测里负样本通常很多，容易出现“简单样本太多、难样本学不到”的问题。
  `Focal Loss` 的思路是：降低简单样本的影响，把注意力更多给难样本。

- 为什么 IoU 类损失常见
  因为检测最关心的是“预测框和真实框重叠得好不好”，IoU 正好直接描述这个问题。

- 面试快答
  检测任务通常联合优化分类损失和回归损失；如果样本不均衡明显，常会用 `Focal Loss`。

## 7. 训练基础：样本分配

- 样本分配是什么
  就是训练时决定“哪些预测位置负责学这个目标，哪些位置不负责”。

- 为什么它很重要
  如果正负样本分配不合理，模型就可能：
  1. 学不到真正该学的位置
  2. 训练不稳定
  3. 收敛慢
  4. 精度差

- 底层逻辑
  训练时一个目标周围可能对应很多候选位置，但不可能让所有位置都去学同一个目标，所以必须规定谁是正样本、谁是负样本。

- 常见思路
  按 IoU 阈值分配、按中心区域分配、按分类质量和回归质量联合分配。

- 通俗理解
  像给学生分作业，得先指定“谁负责哪道题”，不然所有人都抢同一道，或者没人做。

## 8. 样本不均衡和数据增强

- 常见不均衡有哪些
  类别不均衡：某些类很多，某些类很少。
  正负样本不均衡：背景特别多，真正目标很少。
  难样本不足：遮挡、模糊、极端角度样本太少。

- 解决方法怎么记
  重采样、类别加权、`Focal Loss`、难样本挖掘、数据增强。

- 数据增强的底层逻辑
  通过人工改变图像，让模型看到更多样的情况，从而减少过拟合、提升泛化能力。

- 常见增强
  翻转、裁剪、颜色扰动、`Mosaic`、`MixUp`、`Copy-Paste`

- 为什么你面试时答 `Mosaic` 不算全错
  因为它确实是常见增强手段，也可能对少数类有帮助；但更完整的答案应该是“先判断是哪种不均衡，再从重采样、损失函数和增强多个方向一起处理”。

## 9. 分布式训练和普通训练

- 普通训练是什么
  常见就是单卡训练：一张卡处理一个 batch，做前向、反向、更新参数。

- 分布式训练是什么
  多张卡同时训练，每张卡算一部分数据，然后把梯度同步，再统一更新模型参数。

- 底层逻辑
  模型没变，变的是“训练这件事如何分工”。本来一张卡干的活，现在拆给多张卡并行做。

- 为什么要分布式
  训练更快；可以用更大的 batch size；大模型或大数据集更容易跑起来。

- 分布式训练常带来的变化
  学习率常要调；总 batch size 会变大；BatchNorm 可能要用 `SyncBN`；数据加载通常要配分布式采样器。

- 面试快答
  分布式训练本质是多卡数据并行和梯度同步，和单卡训练的主要差别在于速度、总 batch size 和训练配置。

## 10. 多目标跟踪的底层逻辑

- 跟踪到底在做什么
  不是“每一帧重新识别一次”这么简单，而是要回答：当前帧这个框，和上一帧哪个目标是同一个？

- 通用流程
  检测 -> 预测已有目标会出现在哪里 -> 把当前检测结果和已有轨迹做匹配 -> 更新轨迹

- 核心难点
  遮挡、漏检、相似目标太多、目标突然移动、跨镜头切换。

## 11. 卡尔曼滤波

- 是什么
  一种状态估计方法，不是在线训练模型。

- 两步逻辑
  预测：根据上一时刻的位置和速度，猜现在在哪。
  更新：拿当前真实检测到的框来修正这个猜测。

- 为什么跟踪里要它
  因为目标不可能每一帧都检测得完美。即使这一帧检测稍微偏了，或者短暂漏掉，卡尔曼滤波也能给出一个合理的“预测位置”。

- 通俗理解
  像你知道一辆车刚刚往前开，那下一秒大概率也还在往前，而不是瞬间跑到画面另一边。

- 面试快答
  卡尔曼滤波在跟踪里主要用于根据历史状态预测目标下一时刻位置，再用当前检测结果修正预测。

## 12. 匈牙利匹配

- 是什么
  用来解决“多个轨迹和多个检测框应该怎么一一对应”的最优分配问题。

- 底层逻辑
  当一帧里有很多框时，不能靠肉眼拍脑袋说谁对应谁，需要一个算法在整体上找到代价最小的匹配方案。

- 代价矩阵一般怎么来
  可以用位置距离、IoU、外观相似度等构造。

- 通俗理解
  像给学生和座位做最优分配，不是只看一个人，而是看整体安排是否最合适。

## 13. ReID

- 是什么
  `Re-Identification`，重识别。目标是判断两个目标是不是同一个身份。

- 在跟踪里为什么重要
  只看位置不够，因为两个车可能离得很近，或者一个车被挡住后重新出现。此时需要额外看“长相特征”。

- 它怎么工作
  模型会把目标图片映射成一个特征向量 `embedding`。如果两个向量很接近，就说明更可能是同一个目标。

- 常见指标
  `Rank-1`：最像的那个是不是正确身份。
  `Rank-5`：前五个候选里有没有正确身份。
  `mAP`：整体检索排序质量怎么样。

- 通俗理解
  就像给每辆车提一个“外观指纹”，之后拿这个指纹去做匹配。

## 14. DeepSORT 和 ByteTrack

- `DeepSORT`
  核心：运动信息 + 外观信息。
  流程：检测框 -> 卡尔曼滤波预测 -> 提取 ReID 特征 -> 构造代价矩阵 -> 匈牙利匹配。
  适合理解成：既看“它应该出现在哪”，也看“它长得像不像之前那个目标”。

- `ByteTrack`
  核心：不要轻易丢掉低置信度检测框。
  底层逻辑：被遮挡或模糊时，真实目标的检测分数可能下降；如果一刀切把低分框都删了，轨迹就容易断。
  做法：先用高分框匹配，再用低分框补未匹配轨迹。
  适合理解成：它更珍惜那些“分数低但可能是真的”的框。

- 两者区别怎么说
  `DeepSORT` 更强调外观特征和运动信息联合匹配。
  `ByteTrack` 更强调高低分检测框分阶段关联。

- 快速记忆
  `DeepSORT = 卡尔曼滤波 + 匈牙利匹配 + ReID`
  `ByteTrack = 高分框先匹配，低分框再补轨迹`

## 15. 部署基础：ONNX / TensorRT / OM

- `ONNX`
  是什么：一种模型交换格式。
  作用：把训练框架里的模型导出来，方便接到后续部署框架。
  通俗理解：训练和部署之间的“中转站”。

- `TensorRT`
  是什么：NVIDIA 的推理优化和执行引擎。
  作用：解析模型、做图优化、算子融合、支持 `FP16/INT8` 加速。
  通俗理解：让模型在 NVIDIA GPU 上跑得更快的“加速器”。

- `OM`
  是什么：华为昇腾平台常见的离线模型格式。
  常见流程：`pth -> onnx -> om`
  通俗理解：这是昇腾设备能高效执行的那种“最终格式”。

## 16. 量化和 NMS

- 量化是什么
  把高精度数值表示改成低精度表示，比如 `FP32 -> FP16` 或 `INT8`。

- 为什么量化能提速
  因为低精度计算更快、占用内存更少、带宽压力也更小。

- 代价是什么
  可能掉精度，所以通常要做校准、对比测试。

- `NMS` 是什么
  `Non-Maximum Suppression`，非极大值抑制。
  作用：同一个目标可能被预测出多个重叠框，NMS 用来留下最可信的那个，把重复框去掉。

- `NMS` 的逻辑
  按分数排序 -> 先保留最高分框 -> 删除和它重叠太大的其他框 -> 重复这个过程。

- 通俗理解
  像一个目标被多人“重复报名”，最后只保留那个最靠谱的名额。

## 17. 部署优化从哪里做

- 图优化
  常量折叠、算子融合、删除冗余节点、图简化。
  底层逻辑：把多余计算提前消掉，把能合并的步骤合并。

- 精度优化
  `FP16`、`INT8`、混合精度。
  底层逻辑：用更低精度换更高速度和更低资源消耗。

- 工程优化
  固定输入尺寸、静态 `shape`、减少 CPU 和设备之间的数据拷贝、前处理/推理/后处理流水线化、`warmup`。
  底层逻辑：很多时候真正拖慢系统的不只是模型本身，还有数据搬运和流程串行。

- 后处理优化
  优化 `NMS`、减少 Python 层瓶颈、尽量把后处理下沉到设备端。

- 面试快答
  部署优化一般从图优化、量化、内存拷贝、后处理和流水线并行几个方向去做。

## 18. 高频一句话

- `backbone` 提特征，`neck` 做多尺度融合，`head` 输出类别和框。
- `anchor-based` 是先给候选框模板再修正，`anchor-free` 是直接预测目标位置。
- `YOLOv5` 偏成熟工程化，`YOLOX/YOLOv8` 偏新一代 `anchor-free`。
- `DETR` 用 `encoder-decoder + object query` 做集合预测。
- `DINO` 是 `DETR` 改进版，关键点之一是 `denoising training`。
- 检测训练的核心包括损失函数、样本分配、数据增强和训练策略。
- `DeepSORT` 的核心是卡尔曼滤波、匈牙利匹配和 ReID。
- `ByteTrack` 的核心是高低分检测框分阶段关联。
- `ONNX` 是模型中间格式，`TensorRT` 是推理引擎，`OM` 是昇腾部署格式。
- 量化是用低精度换更高速度和更低资源占用。
