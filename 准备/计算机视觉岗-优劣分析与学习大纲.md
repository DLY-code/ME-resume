# 计算机视觉岗-优劣分析与学习大纲

## 1. 这份文档的定位

这份文档不是详细知识手册，而是我面向计算机视觉岗的自我定位、短板判断和学习路线图。

核心目标：

- 明确我现在已经具备哪些可用能力。
- 明确我离计算机视觉岗面试通过还差什么。
- 给出后续复习的优先顺序和知识大纲。

一句话总结：

我当前最大的问题不是“不会用工具”，而是“对工具背后的原理、实验逻辑、问题诊断和技术判断还不够扎实”。

---

## 2. 我当前的优势与短板

### 2.1 当前优势

- 有真实项目经历，不是只做过课程作业。
- 已接触目标检测、多目标跟踪、ReID、边缘部署，方向与视觉算法岗相关。
- 会使用 `PyTorch`、`YOLO`、`ByteTrack`、`DeepSORT`、`ONNX` 等常见工具链。
- 有一定工程落地经验，知道模型训练、推理、部署不是彼此割裂的。
- 上手速度快，借助 AI Agent 能较快把项目推进起来。

### 2.2 当前短板

- 深度学习和计算机视觉基础原理还不够系统。
- 容易停留在“会调用 repo、会让 AI 帮我改代码”的层面。
- 对指标变化、误检漏检、训练不稳定等问题的诊断能力还不够强。
- 对“为什么选这个模型、为什么这样调参、为什么这样改进”的解释不够扎实。
- 项目可以做出来，但未必能在面试里清楚说明自己的判断过程和技术贡献。

### 2.3 当前最核心的能力差距

我需要从下面这种状态：

- 知道怎么把代码跑通
- 知道怎么把模型训出来
- 知道怎么参考现有方案完成一个功能

升级到下面这种状态：

- 能解释模型为什么这么设计
- 能分析模型为什么效果不好
- 能判断问题更可能出在数据、训练、模型还是部署
- 能提出合理而不是拍脑袋的优化方案

优先原则：

## 4. 学习大纲

### 4.1 深度学习基础

#### 核心概念

- 前向传播、反向传播、梯度下降
- 损失函数、优化器、学习率
- `epoch`、`batch size`、`iteration`
- 过拟合、欠拟合、正则化
- `BatchNorm`、`Dropout`

#### 需要达到的程度

- 能用自己的话解释这些概念，而不是只背定义。
- 能说明这些概念在训练中的实际影响。
- 能把训练不收敛、训练慢、过拟合等现象和具体原因联系起来。

#### 面试自查问题

- 为什么学习率太大可能不收敛？
- 过拟合出现时优先从哪些方向处理？
- `BatchNorm` 和 `Dropout` 的作用分别是什么？

### 4.2 计算机视觉基础任务与指标

#### 基础任务

- 图像分类
- 目标检测
- 语义分割
- 实例分割
- 关键点定位
- 多目标跟踪
- ReID

#### 核心指标

- `Precision`
- `Recall`
- `F1-score`
- `IoU`
- `AP / mAP`
- `MOTA`
- `IDF1`
- `Rank-1 / Rank-5`

#### 需要达到的程度

- 能讲清不同任务的输入输出和目标差异。
- 能说明常见指标大致反映什么。
- 能解释“指标提升了”到底提升在哪里。

### 4.3 目标检测

#### 核心内容

- one-stage 与 two-stage
- `YOLO` 系列的基本思路
- `anchor` 与 anchor-free
- 正负样本匹配
- `NMS`
- 小目标问题
- 类别不平衡
- 数据增强

#### 结合项目重点补强

- 为什么选 `YOLOv5`、`YOLOX` 或 `DINO-Swin`
- 速度与精度的取舍
- 数据增强和超参调整对结果的影响
- 漏检与误检的定位思路

#### 面试自查问题

- 为什么 `YOLO` 通常速度更快？
- 漏检和误检分别怎么优化？
- 小目标效果差时优先看什么？

### 4.4 多目标跟踪与 ReID

#### 核心内容

- 检测与跟踪的关系
- 卡尔曼滤波的作用
- 匈牙利算法的作用
- `DeepSORT` 的基本思路
- `ByteTrack` 的基本思路
- `ReID` 的作用
- `ID Switch`
- 遮挡、交叉、丢帧问题

#### 结合项目重点补强

- `DeepSORT` 和 `ByteTrack` 的差异
- 为什么只靠位置不能稳定关联
- 跨镜匹配为什么更难
- 如何分析 ID 切换频繁的问题

#### 面试自查问题

- `ByteTrack` 比 `DeepSORT` 强在哪里？
- 遮挡后目标重新出现，为什么容易跟丢？
- `ReID` 在多目标跟踪里补了什么短板？

### 4.5 数据、标注、实验设计与调参

#### 核心内容

- train / val / test 划分
- 数据泄漏
- 标注规范与标注噪声
- 类别不平衡
- 数据增强适用边界
- ablation study
- 错误分析

#### 需要建立的思维

- 先发现问题，再提出假设，再设计实验验证。
- 不把所有提升都归因给“模型更强”。
- 能区分数据问题、训练问题、模型问题、后处理问题。

#### 面试自查问题

- 你怎么判断提升来自数据还是模型？
- 为什么 train 分数高但 val 分数低？
- 标注不一致会带来什么后果？

### 4.6 PyTorch 训练与调试

#### 核心内容

- `Dataset`
- `DataLoader`
- 模型、损失、优化器、调度器
- 训练循环与验证循环
- checkpoint
- tensor shape 调试
- loss 曲线观察

#### 需要达到的程度

- 能写一个最小训练闭环。
- 能看懂并改动常见训练代码。
- 出现维度错、标签错、loss 不下降时能初步排查。

#### 面试自查问题

- `DataLoader` 在训练里起什么作用？
- 为什么 loss 不下降？
- 维度报错时你怎么定位？

### 4.7 模型部署与推理优化

#### 核心内容

- `PyTorch -> ONNX -> 目标平台`
- 算子不兼容
- 静态 shape 与动态 shape
- 精度对齐
- `FPS`、延迟、吞吐量
- 量化、裁剪、蒸馏

#### 结合项目重点补强

- 为什么部署后可能精度下降
- 为什么边缘设备部署更难
- 如何分析推理速度慢的问题

#### 面试自查问题

- ONNX 是做什么的？
- 部署后精度掉了，你怎么排查？
- 优化推理速度可以从哪几层入手？

### 4.8 项目复盘与面试表达

#### 需要形成的表达框架

- 任务是什么
- 难点是什么
- 我为什么这样选方案
- 我做了哪些实验
- 遇到了什么问题
- 我如何定位问题并改进
- 最终结果是什么
- 下一步还能怎么优化

#### 需要避免的表达方式

- “我主要是调了开源代码。”
- “我用了一个效果更好的模型。”
- “AI 帮我改了代码，能跑就行。”

#### 目标

- 从“我做了什么”升级到“我为什么这么做”。

### 4.9 学习方式与阶段安排

#### 建议学习方式

- 以已有项目为中心反向补原理
- 学一个模块就回到项目中找对应位置
- 对每个改动都记录原因、结果和结论

#### 第一阶段

- 补深度学习基础
- 补视觉任务与指标
- 补 `PyTorch` 最小训练闭环

#### 第二阶段

- 深入检测、跟踪、ReID
- 梳理数据处理、错误分析、调参方法
- 梳理部署链路

#### 第三阶段

- 项目复盘
- 面试问答演练
- 错误案例总结

### 4.10 当前不必优先投入过多时间的内容

#### 可暂时后置

- 复杂数学推导
- `TensorFlow` / `MXNet`
- 3D 检测、点云、BEV
- 大范围追前沿论文
- 非岗位核心的复杂分布式训练

#### 原因

- 当前最缺的不是知识广度，而是项目相关知识的深度和表达能力。
