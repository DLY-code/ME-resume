# 算法岗算法基础速查

目标：打印友好、快速背诵、面试前 5-10 分钟扫一遍。

## 1. 检测器通用结构

- `backbone`：提特征。常见：`ResNet`、`CSPDarknet`、`Swin Transformer`、`MobileNet`
- `neck`：多尺度特征融合。常见：`FPN`、`PAN`、`PAFPN`
- `head`：输出类别和框；常见输出：分类、回归、置信度、IoU
- 一句话：`backbone` 提特征，`neck` 融合特征，`head` 出结果

## 2. anchor-based / anchor-free

- `anchor`：预设候选框，不同尺度、不同长宽比
- `anchor-based`：先设 anchor，再学偏移；代表：`Faster R-CNN`、`YOLOv3`、`YOLOv5`
- `anchor-free`：不设 anchor，直接预测中心点或边界；代表：`FCOS`、`YOLOX`、`YOLOv8`
- 差别：
  `anchor-based` 先验强、设计复杂、要调 anchor；
  `anchor-free` 结构更简洁、超参更少、迁移更方便
- 面试快答：anchor-based 先定义候选框再学偏移，anchor-free 直接预测目标位置

## 3. YOLOv5 / YOLOX / YOLOv8

- `YOLOv5`：成熟、生态好、上手快、常归为 `anchor-based`
- `YOLOX`：`anchor-free`、`decoupled head`、样本分配更先进、速度精度平衡好
- `YOLOv8`：`anchor-free`、结构更现代、使用体验更统一
- `decoupled head`：分类和回归拆成不同分支，减少相互干扰
- 面试快答：YOLOv5 更成熟；YOLOX、YOLOv8 更偏新一代 anchor-free 检测器

### 3.1 卷积基础：YOLO 这类模型到底怎么“看图”

- 你这个理解方向是对的：
  `YOLO` 这类模型主要建立在 `CNN` 这类视觉网络结构上，核心操作就是卷积、下采样、特征融合和检测输出。
  更准确地说：`YOLO` 不是“只有卷积层”，但它的主体确实主要是卷积网络家族。

- 这里和 `backbone / neck / head` 的关系
  `YOLO` 里：
  `backbone` 往往主要由卷积模块堆起来，负责提特征；
  `neck` 继续用卷积、上采样、拼接等操作做多尺度融合；
  `head` 再输出分类和框。
  所以它们不是“不同类型的网络”，而是同一个检测器里按功能划分的三个部分。

## 4. DETR / DINO

- `DETR` 思想：把检测当成集合预测；减少 anchor、NMS 等手工设计
- 流程：图像 -> `backbone` -> 特征图 -> 展平为 `token` + 位置编码 -> `encoder` -> `decoder`(`object queries` + `encoder memory`) -> 分类头/框回归头 -> 类别和框
【图像先经过 backbone 提特征，再把特征图展平成 token 并加入位置编码送入 encoder；decoder 以 object query 为输入，去读取 encoder 输出的全图特征，最后通过预测头输出类别和边界框。】
- `encoder`：做全局建模，建立不同位置间关系
- `decoder`：拿 `object query` 去编码后的特征里“找目标”
- `token`：视觉序列中的一个元素；可以是 patch，也可以是一个特征位置
- `object query`：可学习查询向量；相当于“候选目标槽位”；`num_queries=100` 表示最多预测 100 个目标
- `DETR` 优点：端到端、结构优雅、减少手工设计
- `DETR` 缺点：收敛慢、训练成本高、早期小目标不够好
- `DINO`：`DETR` 改进版；关键点之一是 `denoising training`，帮助更快收敛、更稳定
- `DINO-Swin`：`Swin Transformer` 做 `backbone`，`DINO` 做检测框架
- 面试快答：DETR 用 encoder-decoder 和 object query 做集合预测；DINO 是 DETR 的强化版

### 4.1 transformer划分
- 最稳的划分方式
  视觉特征提取阶段：`图像 -> backbone -> 特征图`
  Transformer 输入准备阶段：`特征图 -> token -> 位置编码`
  Transformer 主体阶段：`encoder -> decoder`
  预测阶段：分类和框

### 4.2 DETR 有没有 backbone / neck / head

- `backbone`
  一定有，负责提图像特征，比如 `ResNet`、`Swin Transformer`。
  ResNet：用卷积提特征
  Swin：把图像切成 patch/token，用分窗口 Transformer 提特征

- `neck`
  原始 DETR 没有像 `FPN/PAN` 那样非常典型、单独拿出来讲的传统 neck。
  因为它把一部分“特征整合、全局关系建模”的工作交给了 Transformer 的 `encoder-decoder`。
  但很多 DETR 改进版会引入多尺度特征处理，这时功能上就会更接近 neck。

- `head`
  也有，只是形式和 YOLO 不太一样。
  `decoder` 输出的是每个 query 的特征，后面还要接分类层和框回归层，才能真正输出类别和边界框。
  所以 DETR 也有检测头，只是通常比较简洁。



## 5. 损失函数

- 作用：衡量预测和真实值差距，指导参数更新
- 检测里常见损失：
  分类损失：`Cross Entropy`、`Focal Loss`
  回归损失：`L1`、`Smooth L1`、`IoU Loss`、`GIoU/DIoU/CIoU`
- `Focal Loss` 适合：类别不均衡、正负样本不均衡、难样本学习
- `IoU` 类损失优点：更贴近检测目标，直接衡量框重叠质量
- 面试快答：检测模型一般联合优化分类损失和框回归损失，不均衡时常用 `Focal Loss`

## 6. 样本分配 / 不均衡 / 数据增强
- 不均衡类型：类别不均衡、正负样本不均衡、难样本不足
- 处理方法：重采样、类别加权、`Focal Loss`、难样本挖掘、数据增强
- 常见增强：翻转、裁剪、颜色扰动、`Mosaic`、`MixUp`、`Copy-Paste`
- 注意：`Mosaic` 是增强手段，不是处理不均衡的唯一答案
- 面试快答：先判断是哪种不均衡，再从重采样、损失函数加权、针对性增强和样本分配优化几个方向处理

## 7. 分布式训练

- 普通训练：单卡跑一个 batch
- 分布式训练：多卡并行；每张卡跑一部分数据；算完梯度后同步，再统一更新参数
- 影响：训练更快、总 batch size 更大、学习率常要调、可能需要 `SyncBN`、分布式采样器
- 面试快答：分布式训练本质是多卡数据并行和梯度同步，和单卡训练的主要差别是速度、batch size 和训练配置

## 8. 多目标跟踪

- 核心问题：这一帧的框和上一帧哪个目标是同一个
- 通用流程：检测 -> 预测轨迹 -> 计算匹配代价 -> 做最优匹配 -> 更新轨迹

### 8.1 卡尔曼滤波

- 不是训练模型，是状态估计方法
- 两步：从 t-1 预测到 t 时刻位置；用 t 时刻的检测结果修正 t 时刻的预测

### 8.2 匈牙利匹配

- 作用：在多轨迹、多检测框之间找整体最优匹配
- 输入：代价矩阵
- 代价常来自：位置距离、IoU、外观相似度

### 8.3 ReID

- 全称：`Re-Identification`
- 作用：把同一目标在不同时间、不同相机下重新认出来
- 输出：外观特征向量 `embedding`
- 意义：解决遮挡、相似目标串 ID、跨镜追踪
- 指标：`Rank-1` 最像的那个对不对；`Rank-5` 前五里有没有对的；`mAP` 看整体排序质量

### 8.4 DeepSORT

- 核心：运动信息 + 外观信息
- 流程：检测框 -> 卡尔曼滤波预测 -> 提取 ReID 特征 -> 构造代价矩阵 -> 匈牙利匹配
- 快答：`DeepSORT = 卡尔曼滤波 + 匈牙利匹配 + ReID`

### 8.5 ByteTrack

- 核心：不要轻易丢掉低分检测框
- 做法：先用高分框匹配已有轨迹，再用低分框补未匹配轨迹
- 优势：遮挡场景更稳、漏检恢复更好
- 和 DeepSORT 区别：
  `DeepSORT` 更强调外观特征；`ByteTrack` 更强调高低分框分阶段关联

## 9. 部署基础

- `ONNX`：模型交换格式；训练框架和部署框架之间的桥梁
- `TensorRT`：NVIDIA 推理优化和执行引擎；做图优化、算子融合、`FP16/INT8` 加速
- `OM`：华为昇腾常用离线模型格式；常见流程：`pth -> onnx -> om`
- 量化：`FP32 -> FP16` 或 `INT8`；优点是更快、更省内存；代价是可能掉精度
- `NMS`：非极大值抑制；作用是去掉同一目标的重复框；流程是按分数排序、保留高分框、删掉高 IoU 框
- `DETR` 常说“不需要 NMS”：因为它本身按集合预测输出目标，重复框问题更少

## 10. 部署优化方向

- 图优化：常量折叠、算子融合、冗余节点删除、图简化
- 精度优化：`FP16`、`INT8`、混合精度
- 工程优化：固定输入尺寸、静态 `shape`、减少数据拷贝、前后处理流水线化、`warmup`
- 后处理优化：优化 `NMS`、把后处理下沉到设备端、减少 Python 层瓶颈
- 面试快答：部署优化通常从图优化、量化、内存拷贝、后处理和流水线并行几个方向做

## 11. 高频一句话

- `backbone` 提特征，`neck` 融合多尺度，`head` 输出类别和框
- `anchor-based` 依赖预设框，`anchor-free` 直接预测目标位置
- `YOLOv5` 偏成熟工程化；`YOLOX/YOLOv8` 偏新一代 anchor-free
- `DETR` 用 `encoder-decoder + object query` 做集合预测
- `DINO` 是 `DETR` 改进版，关键点之一是 `denoising training`
- 检测训练核心：损失函数、样本分配、数据增强、训练策略
- `DeepSORT = 卡尔曼滤波 + 匈牙利匹配 + ReID`
- `ByteTrack` 的核心是高低分检测框分阶段关联
- `ONNX` 是中间格式，`TensorRT` 是推理引擎，`OM` 是昇腾部署格式
- 量化是用低精度换更快速度和更低资源占用
